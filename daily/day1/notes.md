## Podcast Summary

The episode frames the shift from syntax-heavy programming to intent-driven development. Key argument: the bottleneck in software is no longer writing code — it's describing what you want clearly enough for a system to execute it reliably. Vibe coding is the practice of doing exactly that.

---

## Whitepaper Notes — "The New SDLC with Vibe Coding"

### Core Thesis
AI agents are compressing the traditional Software Development Life Cycle. What used to take sprints can now happen in hours when a developer acts as an orchestrator rather than an implementer.

### Factory Model
Developers become system architects who design:
- **Evaluation harnesses** — how do we know the agent did the right thing?
- **Constraint systems** — what is the agent allowed to do?
- **Context harnesses** — what does the agent need to know to execute safely?

### Key Shift
| Before | After |
|--------|-------|
| Write syntax | Describe intent |
| Implement features | Design systems |
| Debug line by line | Evaluate outputs |
| One dev, one task | One dev, many agents |
