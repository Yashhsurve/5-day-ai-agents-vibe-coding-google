## Jun 15 — Day 1

**Mood going in:** Excited but unsure what "vibe coding" actually means in practice.

**Biggest surprise:** The Antigravity IDE genuinely works. I typed what I wanted and got a running app. The snowflakes and balloons app took maybe 5 minutes from prompt to deployed Cloud Run URL. That's wild.

**What clicked:** The "system orchestrator" framing. I've been thinking about AI as a tool I use. This course reframes it — I'm the architect of a system, and the AI is the executor. The SDLC compression isn't about replacing devs, it's about raising the abstraction level.

**What didn't click yet:** How evaluation harnesses actually work in practice. The whitepaper talks about them but I want to see one built.

**Questions I have:**
- How do you debug when the AI writes the code and it breaks?
- What does "context harness" look like in a real production codebase?

---

## Jun 16 — Day 2

**Mood going in:** Still processing Day 1. Ready for more.

**Biggest surprise:** MCP is simpler to add than I expected. One configuration and Antigravity suddenly had access to all of Google's public developer docs in real-time.

**What clicked:** The protocol stack vision. MCP + A2A + A2UI + AP2/UCP isn't just tooling — it's an emerging standard for how agents plug into the world. If this becomes the norm, building agents becomes more like assembling Lego bricks than writing custom glue code.

**What I shipped:** A Flask app that fetches BigQuery Release Notes from a live XML feed and lets you tweet any update. Built entirely via CLI prompt. No manual coding.

**What didn't click yet:** A2UI — generative UI sounds powerful but I haven't seen a real example yet.

**Questions I have:**
- How do you handle rate limits and failures in MCP connections at scale?
- What happens when two A2A agents disagree?
